# ST / *ST 退市风险预测规则手册

依据 2025 年修订版上市规则（上交所主板/科创板、深交所主板/创业板）。
官方文件存放于 `docs/` 目录。

---

## 一、*ST（退市风险警示）— 财务类

触发任一即实施 *ST，**最近一个会计年度**数据：

### 条件 1：利润+营收不达标

| 板块 | 利润指标 | 营收门槛 |
|------|---------|---------|
| 上交所主板 | 利润总额 / 净利润 / 扣非净利润 **三者孰低 < 0** | 扣除后营收 < **3 亿** |
| 深交所主板 | 同上 | 扣除后营收 < **3 亿** |
| 科创板 | 同上 | 扣除后营收 < **1 亿** |
| 创业板 | 同上 | 扣除后营收 < **1 亿** |

- "扣除后营收" = 营业收入 - 与主营无关的收入 - 不具商业实质的收入
- 代码中对应字段：`total_profit`（利润总额）、`n_income_attr_p`（净利润）、`profit_dedt`（扣非净利润）、`revenue`（营收）

**实现状态：✅ 已实现**
- 文件：`backend/app/main.py` → `get_st_predict()`
- **Group A**：已发 2025 年报的用 `income`(end_date=20251231) 实际数据判断
- **Group B**：未发 2025 年报的用 `forecast`(续亏/首亏预告) + 2024 `income` 营收推算
- 门槛：主板 3 亿，创业板/科创板/北交所 1 亿

### 条件 2：期末净资产为负

所有板块一致：最近一个会计年度经审计的期末净资产 < 0。

- 代码中对应字段：`bps`（每股净资产），来自 `fina_indicator` 表

**实现状态：✅ 已实现**

### 条件 3：审计意见

所有板块一致：年报被出具**无法表示意见**或**否定意见**的审计报告。

**实现状态：❌ 未实现**
- 原因：Tushare 无独立的审计意见数据接口，需从年报 PDF 或公告中提取
- 后续方案：可对接巨潮资讯网公告解析，或等 Tushare 新增接口

### 条件 4：追溯重述触发

追溯重述后触及条件 1 或条件 2。

**实现状态：❌ 未实现**
- 原因：追溯重述属事后调整，无法提前预测；发生后依赖交易所公告
- 后续方案：监控交易所公告中"追溯重述"关键词

### 条件 5：证监会处罚导致触及

行政处罚揭示年报虚假记载，导致实际触及条件 1 或条件 2。

**实现状态：❌ 未实现**
- 原因：依赖证监会处罚公告，属突发事件
- 后续方案：监控证监会处罚公告

### 科创板特殊豁免

研发型公司（第五套标准上市）自上市之日起**前 3 个完整会计年度**不适用条件 1、2、4。

**实现状态：❌ 未实现**
- 原因：需要知道每只科创板股票的上市标准（第几套），Tushare 无此字段
- 后续方案：手动维护一张科创板第五套标准上市的股票清单

## 二、*ST — 规范类

| 条件 | 门槛 | 各板一致 |
|------|------|---------|
| 年报/半年报未按期披露 | 停牌 2 个月仍未披露 | ✓ |
| 半数以上董事不保真 | 停牌 2 个月仍如此 | ✓ |
| 财报重大差错未改正 | 证监会责令 + 停牌 2 个月未改 | ✓ |
| 控股股东资金占用（严重） | ≥ 净资产 **30%** 或 ≥ **2 亿**，停牌 2 月未改 | ✓ |
| 连续 2 年内控否定/无法表示 | 连续 2 个会计年度 | ✓ |

**实现状态：❌ 全部未实现**
- 原因：这类条件依赖交易所公告/证监会处罚等外部事件，无法从财务数据预测
- 后续方案：接入交易所公告监控，按关键词匹配

## 三、ST（其他风险警示）

### 各板一致的条件

| 条件 | 门槛 | 实现状态 |
|------|------|---------|
| 资金占用/违规担保 | ≥ 净资产 5% 或 ≥ 1000 万，1 月未整改 | ❌ 依赖公告 |
| 董事会/股东会失灵 | 无法正常召开会议 | ❌ 依赖公告 |
| 内控审计否定/无法表示 | 最近 1 个会计年度 | ❌ 无数据源 |
| 经营中断 | 预计 3 个月恢复不了 | ❌ 依赖公告 |
| 银行账户冻结 | — | ❌ 依赖公告 |
| 连续 3 年亏损 + 持续经营疑虑 | 扣非前后孰低均负 + 审计有疑虑 | ❌ 缺审计意见 |
| 财务造假(未达重大违法) | — | ❌ 依赖处罚公告 |

### 分红不达标（铁公鸡条款）

前提：最近一年净利润为正 **且** 未分配利润为正。

| 板块 | 分红门槛 | 研发豁免 |
|------|---------|---------|
| 上交所主板 | 3 年累计 < 年均利润 30% **且** < **5000 万** | **无** |
| 深交所主板 | 同上 | **无** |
| 科创板 | 3 年累计 < 年均利润 30% **且** < **3000 万** | 研发 > 15% 营收 或 > 3 亿 |
| 创业板 | 同上 | 研发 > 15% 营收 或 > 3 亿 |

补充规则：
- 以现金回购并注销的金额计入现金分红
- 上市不满 3 个完整会计年度的，以上市后首个完整年度起算
- 将未分配利润全部分配后仍触及的，不实施 ST

**实现状态：❌ 未实现**
- 原因：需要分红数据（Tushare `dividend` 接口）+ 研发投入数据
- 后续方案：拉取 `dividend` 表，结合 `fina_indicator` 的研发投入字段计算

## 七、板块判断规则

```python
code = rec["ts_code"]
# 创业板(30xxxx) / 科创板(688xxx) / 北交所(.BJ) → 1亿
# 主板(60xxxx / 00xxxx) → 3亿
is_small_board = code.startswith("3") or code.startswith("688") or code.endswith(".BJ")
threshold = 1e8 if is_small_board else 3e8
```

## 八、注意事项

1. forecast 表的 `net_profit_min/max` 单位是**万元**，income 表的 `n_income_attr_p` 单位是**元**，显示前需统一
2. 预计 ST 日 = 年报披露日（`pre_date`）之后的**第一个交易日**，从 `trade_cal` 查（范围 >= 当年初，不限 >= today）
3. 已被 ST 的股票（在 `stock_st` 最新日期中）需排除
4. 有扭亏/预增/续盈/略增预告的股票需排除
5. 查询基于**当前年报披露季的年度数据**（如2026年4月 → 用2025年报数据），不是固定2024
6. 预告类型同时覆盖**续亏**和**首亏**（首亏也意味着净利润为负）
