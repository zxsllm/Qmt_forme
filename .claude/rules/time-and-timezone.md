# 时间与时区约定

## 硬约束

- **所有时间一律用本地 CST（北京时间，UTC+8）**，不得用 UTC、不得换算、不得说"UTC 现在是 X 点"或"按 UTC X 时差"。
- 输出"几点开盘""距离 X 还有几小时"时，直接用墙上时间，不加 UTC 解释。
- 涉及 cron / 定时任务 / 日志时间戳，统一 `Asia/Shanghai`。
- A 股交易时刻表（默认）：09:30 开盘 / 11:30 午休 / 13:00 续盘 / 14:55 尾盘窗口 / 15:00 收盘。
- "今天" = 用户本地日历日，跟系统提示的 `currentDate` 一致即可，不要再做时区换算。

## 日期口径（沿用全局规则）

- "今天" 用 `getFullYear() / getMonth() + 1 / getDate()`，**禁** `toISOString().slice(0,10)`。
- 日期差 = 先构本地 `Date`、整日差，不要 `Date.parse` 相减。
- 后端 `datetime` 默认本地 naive；存 PG 也按默认（不带时区）。

## 为什么

- 用户在 GMT+8 工作，交易日历 / 盘中时刻 / 复盘文本都按本地解读。
- 切换 UTC 会让"9:30 开盘""现在 1:58"变成换算题，浪费心智。
- 历史教训：曾因 `toISOString` 把"今天"算成昨天，前端日期差错位。
