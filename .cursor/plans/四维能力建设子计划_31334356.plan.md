# 四维能力建设子计划 (Phase 4.8)

> 定位：Phase 5 (QMT实盘) 前的中间能力层，为各类交易模式提供结构化数据基座。
> 优先级：消息面 > 基本面 > 情绪面 > 技术面

---

## A. 消息面 Stage 1 — 现有新闻/公告清洗与分类

**现状问题**:

- `stock_news` 表只有 `datetime, content, channels, source`，无分类、无关联股票、无情绪标签
- 个股新闻靠 `content ILIKE '%股票名%'` 匹配，漏报误报严重
- 盘中/盘后消息未区分，无法针对性使用
- 公告 `stock_anns` 仅有 `ts_code, ann_date, title, url`，无类型分类

### A1-Step1: 新闻分类表 + 规则引擎

新增 `news_classified` 表（与 `stock_news` 一对一关联）：

```python
class NewsClassified(Base):
    __tablename__ = "news_classified"
    news_id: int          # FK -> stock_news.id
    news_scope: str       # "macro" / "industry" / "stock" / "mixed"
    time_slot: str        # "intraday" / "after_hours" / "pre_open"
    related_codes: str    # JSON array: ["000001.SZ", "600519.SH"]
    related_industries: str  # JSON array: ["电子", "新能源"]
    sentiment: str        # "positive" / "negative" / "neutral"
    keywords: str         # 提取的关键词 JSON
    classified_at: datetime
```

规则引擎 `backend/app/shared/news_classifier.py`：

- **scope分类**: 宏观关键词库 (央行/国务院/利率/GDP/CPI/MLF/LPR...)、行业关键词映射 (来自 `index_classify` 申万行业树)、个股代码/名称提取 (正则 + `stock_basic` 名称表匹配)
- **时间分类**: 对比 `datetime` 与交易日历 + 交易时段 (09:30-15:00为盘中，其余为盘后，08:00-09:30为盘前)
- **情绪分类**: A股正面/负面关键词库 (涨停/突破/超预期/景气/利好 vs 跌停/暴雷/退市/利空/违规)

### A1-Step2: 公告分类增强

增强 `stock_anns` 或新增 `anns_classified`：

- **公告类型**: 业绩预告/业绩快报/分红/增减持/股权变动/定增/回购/违规处罚/其他
- 基于 `title` 关键词匹配分类

### A1-Step3: 历史数据回填 + 实时管道

- 批量分类已有 `stock_news` 记录 (脚本 `scripts/classify_news.py`)
- 集成到 `scheduler._pull_news_sync()` 流程，新闻入库后自动分类
- 集成到 `pull_anns.py` 流程

### A1-Step4: API + 前端升级

- 已实现分类新闻/公告 REST：`/api/v1/market/news/classified`、`/api/v1/market/anns/classified`、`/api/v1/market/news/stats`（支持 scope/time_slot/sentiment 等筛选）
- 预留 `/api/v1/news/daily-brief` 每日消息摘要 (按类型聚合)，待接
- 前端 `SidebarNews.tsx` 仍为快讯区；**分类消息浏览**独立页 `NewsPage.tsx`，路由 `/news`（筛选 + 统计卡片 + 分类公告 Tab）
- **盘前消息看板**（昨日盘后+今日盘前聚合）仍为后续增强项

### A1-Step5: 分类质量迭代

- 提供人工标注/纠正UI（前端简单标记 correct/wrong）
- 根据纠正反馈优化关键词库
- 预留大模型分类接口（之后可对接 DeepSeek 等）

## B. 基本面能力

**现状**: 仅有 `daily_basic` (PE/PB/市值截面) + `index_classify` (行业分类树) + `sw_daily` (行业指数)。无财报数据。

### B-Step1: 核心财务数据入库

新增3张核心表 + Tushare封装 + 拉取脚本：


| 新表               | Tushare接口        | 用途                | 备注            |
| ---------------- | ---------------- | ----------------- | ------------- |
| `fina_indicator` | `fina_indicator` | ROE/净利率/毛利率/营收增速等 | 最核心，一张表浓缩关键指标 |
| `income`         | `income`         | 利润表(营收/净利润/扣非)    | 深度分析用         |
| `forecast`       | `forecast`       | 业绩预告(预增/预减/扭亏)    | 事件驱动核心        |


**拉取策略**: 优先科技+新能源行业 (~800只)，通过 `index_classify` 的申万分级筛选 `industry_name` 属于目标行业的 `ts_code`。

### B-Step2: 概念板块同步补全

- 新建 `scripts/pull_concept.py`，拉取 `concept_list` + `concept_detail`
- 注册到 `daily_sync.py`（概念成分变动较少，可周频更新）

### B-Step3: 公司画像数据


| 新表                | Tushare接口         | 用途               |
| ----------------- | ----------------- | ---------------- |
| `fina_mainbz`     | `fina_mainbz`     | 主营业务构成 (产品/地区拆分) |
| `disclosure_date` | `disclosure_date` | 财报披露日期 (事件日历)    |


### B-Step4: 行业画像 + 公司筛选器

- `backend/app/shared/fundamental.py`：
  - `industry_profile(industry_code)` → 行业内所有公司的 ROE/增速/市值分布
  - `company_score(ts_code)` → 业绩纯度 (主营占比) + 成长性 (营收增速) + 弹性 (流通市值) 综合评分
  - `event_calendar(start, end)` → 财报披露 + 业绩预告日期汇总
- API: `/api/v1/fundamental/industry/{code}`, `/api/v1/fundamental/company/{ts_code}`
- 前端: 基本面分析页 (行业概览 + 公司画像)；当前已预留路由 `/fundamental` → `FundamentalPage.tsx`（占位，待本步实现）

## C. 情绪面能力

**现状**: 有 `stock_limit` (涨跌停价) 和 `concept_list/detail` (概念板块)，但缺少打板专题数据。Tushare 打板专题提供丰富API。

### C-Step1: 涨跌停榜单数据


| 新表               | Tushare接口 | 数据内容                     |
| ---------------- | --------- | ------------------------ |
| `limit_list_ths` | 同花顺涨跌停榜单  | 每日涨停/跌停/炸板个股列表+首板时间+封单金额 |
| `limit_stats`    | 涨跌停和炸板数据  | 每日涨停家数/跌停家数/炸板家数/封板率     |


- 新建 `TushareService.limit_list_ths()`, `limit_stats()`
- 新建 `scripts/pull_limit_board.py`
- 注册到 `daily_sync.py`

### C-Step2: 连板天梯 + 龙虎榜


| 新表           | Tushare接口  | 数据内容                        |
| ------------ | ---------- | --------------------------- |
| `limit_step` | limit_step | 每日各连板层级股票 (2连板/3连板/.../最高板) |
| `top_list`   | top_list   | 龙虎榜上榜股票 + 买卖金额              |
| `hm_detail`  | hm_detail  | 知名游资席位交易明细                  |


### C-Step3: 板块情绪指标


| 新表               | Tushare接口      | 数据内容         |
| ---------------- | -------------- | ------------ |
| `limit_cpt_list` | limit_cpt_list | 每日涨停股最多的概念板块 |
| `dc_hot`         | dc_hot         | 市场关注度排名      |


### C-Step4: 情绪面分析引擎

`backend/app/shared/sentiment.py`：

- `market_temperature(trade_date)` → 涨停家数/跌停家数/封板率/连板高度/炸板率 → 综合情绪温度
- `board_leader(trade_date, concept)` → 板块内龙1/龙2识别 (按首封时间排序)
- `continuation_analysis(ts_code)` → 连板分析 (当前连板天数/历史最高/断板概率)
- `hot_money_signal(trade_date)` → 知名游资买入信号

### C-Step5: 盘前计划系统

`backend/app/shared/premarket.py`：

- 结合**盘后消息** (A1分类结果) + **涨停数据** (C1-C3) + **基本面** (B)
- 每日盘前生成：
  - 昨日市场情绪总结 (涨停家数/连板高度/强势板块)
  - 今日关注列表 (昨日龙头+连板股/消息催化+基本面支撑)
  - 风险警示 (昨日炸板股/利空消息股)
- API: `/api/v1/premarket/plan`
- 前端: 新增 **盘前计划** 页面（与当前侧边栏「消息中心/情绪看板」路由独立，后续可再聚合入口）

## D. 技术面 (简化版)

**定位**: 辅助风控，不做复杂策略。聚焦价格+成交量两个核心维度。

### D-Step1: 关键技术信号检测

`backend/app/shared/tech_signal.py`：

- `consecutive_limit_count(ts_code, trade_date)` → 当前连板天数
- `volume_anomaly(ts_code, trade_date)` → 放量/缩量倍数 (vs 20日均量)
- `gap_analysis(ts_code, trade_date)` → 跳空缺口检测
- `support_resistance(ts_code, days=60)` → 简单支撑/压力位 (近期高低点)

### D-Step2: 风控辅助规则

- 连板断板预警 (N连板后缩量/放量阴线 → 卖出信号)
- 天量见天价预警 (成交量创N日新高 + 长上影线)
- 集成到模拟盘 `risk/` 模块作为辅助指标

## 架构原则

- 所有新 Tushare API 调用走 `TushareService` 封装
- 所有新表用 Alembic 迁移管理
- 所有拉取脚本注册到 `daily_sync.py`
- 所有分析引擎放 `backend/app/shared/` (消息/基本面/情绪/技术共享层)
- API 统一挂到 `backend/app/main.py` 或独立 router
- 前端复用现有布局 + AntD5 组件体系；交易相关已合并为 `/trading`、`/strategy`、`/system`，四维专题页为 `/news`、`/sentiment`、`/fundamental`（见路由重构计划）
